# devops-capstone-project

En este proyecto Capstone, aplicarás muchas de las tecnologías y conceptos que aprendiste en los cursos anteriores para construir y entregar un microservicio de Cuentas de Clientes completamente funcional.

Comenzando en Módulo 1, iniciarás este viaje del capstone desarrollando un plan ágil para construir tu microservicio RESTful. Crearás un repositorio de GitHub y un tablero Kanban para gestionar el proyecto, construirás una plantilla de historias de usuario para redactar historias bien estructuradas y poblarás tu Product Backlog con todas las historias necesarias para implementar el microservicio de Cuentas de Clientes. Después de completar tu backlog, prepararás un plan de sprint configurando sprints, estimando puntos de historia, asignando historias al sprint correspondiente y construyendo tu Sprint Backlog. A lo largo del Capstone, tomarás capturas de pantalla y registrarás URLs de GitHub como evidencia para la entrega final.

En Módulo 2, comenzarás Sprint 1 configurando tu entorno de proyecto y desarrollando el microservicio de Cuentas de Clientes utilizando desarrollo guiado por pruebas (TDD). A medida que trabajes en cada historia, la moverás a través de tu tablero Kanban, de “Backlog” a “En Progreso”, luego a “Hecho” y, finalmente, a “Cerrado”. Crearás una rama de desarrollo para tu trabajo de sprint y subirás cambios a GitHub mediante la presentación de solicitudes de extracción. Escribirás casos de prueba para las funciones de lectura, actualización, eliminación y listado de tu servicio RESTful Flask y escribirás solo el código necesario para que cada prueba pase. Ejecutarás nosetests para asegurarte de que todas las pruebas pasen y utilizarás la herramienta de cobertura para mantener al menos un 95% de cobertura de prueba.

En Módulo 3, pasarás a Sprint 2, donde configurarás un flujo de trabajo de integración continua (CI) de GitHub Actions que se ejecute automáticamente cada vez que se realice una solicitud de extracción o un push a la rama principal. Como parte del Sprint 2, construirás un flujo de trabajo que realice análisis de código con Flake8, ejecute tus pruebas, verifique la cobertura de código y valide la calidad del código. Luego, mejorarás tu microservicio agregando prácticas de codificación segura. Esto incluye agregar Flask-Talisman para los encabezados de seguridad y Flask-CORS para establecer políticas de Compartición de Recursos de Origen Cruzado. Siguiendo TDD, escribirás pruebas que fallen, implementarás las características de seguridad requeridas, asegurarás que las pruebas pasen y fusionarás tu trabajo en la rama principal.

En Módulo 4, comenzarás Sprint 3, donde trabajarás en historias de usuario relacionadas con el despliegue. Crearás un Dockerfile, construirás una imagen de Docker para el servicio de Cuentas de Clientes y subirás la imagen al Registro de Contenedores de IBM Cloud. Desplegarás manualmente tu aplicación en un clúster de OpenShift/Kubernetes. También crearás un servicio de PostgreSQL en OpenShift y escribirás los manifiestos YAML de despliegue y servicio de Kubernetes necesarios para desplegar tu microservicio. Al igual que en los sprints anteriores, cometerás, subirás y fusionarás tus cambios a medida que avances en cada historia.

En Módulo 5, extenderás tu trabajo de despliegue creando un pipeline de entrega continua (CD) de Tekton para automatizar el despliegue de tu microservicio en Kubernetes. Cuando se active, este pipeline clonará tu repositorio, analizará y probará tu código, construirá tu imagen de Docker y desplegará el servicio actualizado en el clúster sin intervención manual.

En Módulo 6, recopilarás y organizarás toda la evidencia requerida—capturas de pantalla, URLs y salidas—de los laboratorios prácticos para la entrega. Tu trabajo será evaluado utilizando Opción 1: Entrega y Evaluación Calificada por IA u Opción 2: Entrega y Evaluación Calificada por Pares.

Finalmente, en Módulo 7, completarás el Examen Final para validar tu comprensión de los conceptos y prácticas de DevOps aplicados a lo largo del proyecto Capstone.
